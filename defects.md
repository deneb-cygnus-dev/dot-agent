# `.claude/` Defect Report

> **Scope:** `.claude/` (symlink → `.agent/` git submodule)
> **Date:** 2026-03-03
> **Inspector:** Claude Code

---

## CRITICAL

### 1. Missing `scripts/` directory — 10 hook scripts don't exist

All of the following files are referenced in `hooks/hooks.json` but the entire `scripts/` directory is absent from `.agent/`:

```text
scripts/hooks/check-console-log.js
scripts/hooks/doc-file-warning.js
scripts/hooks/evaluate-session.js
scripts/hooks/post-edit-console-warn.js
scripts/hooks/post-edit-format.js
scripts/hooks/post-edit-typecheck.js
scripts/hooks/pre-compact.js
scripts/hooks/session-end.js
scripts/hooks/session-start.js
scripts/hooks/suggest-compact.js
```

**Affected hooks:**

| Hook event | Hook description |
|---|---|
| `PreToolUse` (`Write`) | Doc file warning |
| `PreToolUse` (`Edit\|Write`) | Suggest compact |
| `PostToolUse` (`Edit`) | Auto-format JS/TS |
| `PostToolUse` (`Edit`) | TypeScript check |
| `PostToolUse` (`Edit`) | console.log warning |
| `Stop` | Console.log audit |
| `PreCompact` | Save state before compaction |
| `SessionStart` | Load previous context / detect package manager |
| `SessionEnd` | Persist session state |
| `SessionEnd` | Evaluate session for extractable patterns |

Every one of these hooks will error at runtime. Because hook errors are non-fatal by default, they will silently fail — no formatting, no TypeScript checking, no session persistence.

## HIGH

### 2. `${CLAUDE_PLUGIN_ROOT}` is undefined in project-level context

All 10 missing scripts are referenced via the pattern `${CLAUDE_PLUGIN_ROOT}/scripts/hooks/<name>.js`. This environment variable is **only injected by Claude Code for plugins installed under `~/.claude/plugins/`**. When `.agent` is loaded as a project-level `.claude/` directory (via symlink), `${CLAUDE_PLUGIN_ROOT}` is never set.

This means even if the scripts were present, the path expansion would produce an empty or broken path and every hook using this variable would still fail.

**Two affected hook groups:**

1. The 10 script-based hooks listed above (blocked by missing scripts + undefined var).
2. The inline Node.js hooks (git push reminder, PR logger, tmux reminder, build analysis) do **not** use `${CLAUDE_PLUGIN_ROOT}` and are unaffected.

**Possible fixes:**

- Install `.agent` as a proper Claude plugin (`~/.claude/plugins/dot-agent/`) so the variable is set automatically.
- Replace `${CLAUDE_PLUGIN_ROOT}` with an absolute path injected via a startup script or shell profile (`export CLAUDE_PLUGIN_ROOT=/path/to/.agent`).
- Inline the logic from the missing scripts directly into `hooks.json` as Node.js one-liners (consistent with the existing inline hooks).

## MEDIUM

### 3. `mise.toml` Node version conflicts with main project

| File | Node version |
|---|---|
| `.agent/mise.toml` | `25.7.0` (non-LTS, cutting-edge) |
| `mise.toml` (project root) | `22.22.0` (LTS, matches `CLAUDE.md`) |

`mise` resolves the nearest `mise.toml` walking up from the current directory. Any work done inside the `.agent/` directory (e.g., running `install.sh`, linting markdown) will pick up Node 25.7.0 instead of the project-pinned 22.22.0. This creates an inconsistency with the stated toolchain in `CLAUDE.md`.

The Go version (`1.25.6`) is identical in both files and is not a concern, but pinning Go in a configuration-only repository serves no functional purpose.

**Fix:** Remove `go` from `.agent/mise.toml` and align `node` to `22.22.0`.

## LOW

### 4. Wrong `.gitignore` for a Claude Code configuration repo

`.agent/.gitignore` is a Go project template. It excludes artifacts that will never appear in a configuration-only repository:

```text
*.exe  *.exe~  *.dll  *.so  *.dylib   # Go binaries
*.test                                  # Go test binaries
*.out  coverage.*  *.coverprofile      # Go coverage
go.work  go.work.sum                   # Go workspace
```

None of these files will ever be generated by `.agent/`. Conversely, the `.gitignore` doesn't exclude things that could legitimately appear in a config repo (e.g., `node_modules/`, OS metadata like `.DS_Store`, or temporary output from `install.sh`).

**Fix:** Replace with a config-repo-appropriate `.gitignore`:

```gitignore
# OS
.DS_Store
Thumbs.db

# Temp files from install.sh
/tmp/

# Node (if scripts are added)
node_modules/
```

## INFO

### 5. No `settings.json` in `.claude/`

There is no `.claude/settings.json` (resolved path: `.agent/settings.json`). Hooks are placed in `hooks/hooks.json`, which is the correct layout for a Claude Code plugin directory. However, project-level configuration that normally lives in `settings.json` — such as `allowedTools`, `model`, `apiKeyHelper`, or `trustedExecutables` — has no home at the project level.

This is not currently breaking anything, but it means any project-level Claude Code settings must be configured globally in `~/.claude/settings.json` rather than per-project.
